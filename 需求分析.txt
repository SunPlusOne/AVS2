1. 项目概述
1.1 项目背景
视频发声物体分割（Audio-Visual Segmentation，AVS）是计算机视觉与音频信号处理交叉领域的前沿研究方向。传统视频分割方法主要依赖颜色、纹理及运动矢量等视觉特征，在面对复杂光照变化、目标遮挡或背景纹理相似等场景时，难以精准定位与当前事件相关的目标物体。
本系统通过引入音频模态作为视觉信息的补充，融合视听多模态特征，实现对视频中正在发声物体的像素级精准分割，打破了传统视觉分割的局限性，具有重要的理论意义与实际应用价值。
1.2 系统目标
•	实现对视频中发声物体的像素级精准分割，支持单声源与多声源场景
•	集成 AVSegFormer、VCT、COMBO 三种先进算法，提供灵活的模型选择
•	提供友好的 Web 前端界面（基于 Vue.js），支持视频上传、结果预览与下载
•	构建稳健的 Python 后端服务（基于 FastAPI），承载数据预处理、模型推理与结果生成
•	支持管理员进行模型权重管理与系统状态监控
1.3 技术架构概览

层次	技术选型	主要职责
前端层	Vue 3 + Vite + Element Plus	用户交互界面、视频上传、结果可视化
通信层	RESTful API / WebSocket	前后端数据交互、推理进度实时推送
后端层	Python + FastAPI	业务逻辑、任务调度、文件管理
算法层	PyTorch + AVSegFormer/VCT/COMBO	视听特征提取、模型推理、掩码生成
媒体处理	FFmpeg + Librosa + OpenCV	视频解封装、音频提取、帧序列合成
存储层	本地文件系统	视频文件、模型权重、临时缓存、日志

2. 用户需求分析
2.1 用户角色定义

角色	描述	主要操作
普通用户	使用系统进行视频分割的终端用户	上传视频、选择算法、查看/下载结果
管理员	负责系统维护与模型管理的运维人员	上传模型权重、查看日志、管理算法库

2.2 用户故事
2.2.1 普通用户
•	作为普通用户，我希望能够上传本地视频文件，以便对视频中的发声物体进行分割分析。
•	作为普通用户，我希望能够选择不同的分割算法，以便在精度与速度之间做出权衡选择。
•	作为普通用户，我希望能够实时查看任务处理进度，以便了解当前推理状态。
•	作为普通用户，我希望能够对比查看原始视频与分割结果，以便直观评估分割效果。
•	作为普通用户，我希望能够下载分割结果视频或逐帧掩码文件，以便后续使用。
2.2.2 管理员
•	作为管理员，我希望能够上传新版本的模型权重文件，以便更新系统中的算法模型。
•	作为管理员，我希望能够查看系统运行日志，以便监控性能瓶颈与排查异常错误。
•	作为管理员，我希望能够管理算法元数据配置，以便向用户展示准确的算法描述信息。

3.1 功能模块总览

模块编号	模块名称	所属端	优先级
M01	用户交互模块	前端	高
M02	数据管理与预处理模块	后端	高
M03	算法管理模块	后端（管理员）	中
M04	模型推理模块	后端	高
M05	结果后处理模块	后端	高

M08	系统日志与状态管理模块	后端（管理员）	中

3.2 前端功能需求（Vue.js）
3.2.1 M01 — 用户交互模块
本模块提供系统的主界面，是用户与系统交互的入口。
•	FR-01-01：提供视频文件拖拽上传控件，支持 MP4、AVI、MOV、MKV 等主流格式，单文件大小不超过 2GB。
•	FR-01-02：提供算法选择下拉框，列出 AVSegFormer、VCT、COMBO 三种算法及其适用场景说明。
•	FR-01-03：提供任务启动按钮，点击后触发后端推理请求，按钮在任务执行期间置灰并显示加载状态。
•	FR-01-04：以进度条形式实时展示任务处理进度（0%–100%），通过 WebSocket 或轮询接口获取进度信息。
•	FR-01-05：提供任务取消功能，允许用户在推理过程中中断当前任务。
3.2.2 M06 — 可视化展示模块
本模块负责在前端渲染并展示分割结果。
•	FR-06-01：支持左右分屏对比模式，左侧播放原始视频，右侧同步播放分割结果视频，双侧播放器时间轴联动。
•	FR-06-02：支持画中画叠加模式，将分割掩码半透明叠加在原始视频上显示，提供透明度调节滑块。
•	FR-06-03：提供视频播放基础控制：播放/暂停、倍速切换（0.5x / 1x / 2x）、进度条拖拽定位。
•	FR-06-04：支持逐帧查看模式，可通过键盘方向键或帧步进按钮精确浏览每一帧分割效果。
3.2.3 文件下载交互
•	FR-07-01：分割完成后显示下载面板，提供【下载结果视频（MP4）】与【下载逐帧掩码（ZIP）】两种选项。
•	FR-07-02：提供历史任务列表页面，展示已完成任务的缩略图、算法名称、处理时间与下载入口。
3.3 后端功能需求（Python / FastAPI）
3.3.1 M02 — 数据管理与预处理模块
•	FR-02-01：接收前端上传的视频文件，对文件格式与完整性进行校验，返回校验结果。
•	FR-02-02：调用 FFmpeg 对视频进行解封装，分离视频帧序列（PNG/JPEG）与音频轨道（WAV）。
•	FR-02-03：对视频帧进行尺寸归一化（统一 resize 至模型输入尺寸，如 224×224 或 384×384）。
•	FR-02-04：使用 Librosa 对音频进行重采样（统一至 16kHz）并提取 Log-Mel 频谱图特征。
•	FR-02-05：确保视频帧与音频特征在时间戳上的严格同步对齐，以帧为单位组织数据对。
3.3.2 M04 — 模型推理模块
•	FR-04-01：提供统一的推理接口（Inference Interface），内部适配 AVSegFormer、VCT、COMBO 三种算法。
•	FR-04-02：根据请求参数动态加载对应算法的预训练权重（.pth 文件），支持 GPU 加速推理（CUDA）。
•	FR-04-03：以异步任务方式执行推理（使用 Celery 或 FastAPI BackgroundTask），避免阻塞 HTTP 请求。
•	FR-04-04：支持批量帧推理，合理管理显存（VRAM），避免 OOM（显存溢出）错误。
•	FR-04-05：推理过程中通过 WebSocket 实时向前端推送当前进度（已处理帧数 / 总帧数）。
3.3.3 M05 — 结果后处理模块
•	FR-05-01：对模型输出的原始 Logits 执行 Sigmoid 激活与二值化阈值处理（默认阈值 0.5）。
•	FR-05-02：对二值掩码执行连通域分析（cv2.connectedComponentsWithStats），过滤面积过小的噪点区域。
•	FR-05-03：将单通道二值掩码转换为 RGB 颜色掩码（可配置颜色，默认红色高亮），并叠加至原始帧。
•	FR-05-04：调用 FFmpeg 将逐帧合成图像编码为 MP4 视频文件，保留原始音频轨道。
3.3.4 M03 — 算法管理模块（管理员）
•	FR-03-01：提供模型权重文件上传接口，支持 .pth 格式，上传后自动注册至算法配置文件。
•	FR-03-02：支持配置算法元数据（算法名称、版本号、适用场景描述、输入尺寸要求）。
•	FR-03-03：支持对已注册模型进行版本切换（激活/禁用）与删除操作。
3.3.5 M08 — 系统日志与状态管理模块
•	FR-08-01：记录用户每次操作行为日志，包含时间戳、操作类型、选用算法、输入文件信息。
•	FR-08-02：记录每次推理任务的耗时（总时长、每帧平均耗时）与显存占用峰值。
•	FR-08-03：记录异常错误日志（OOM、格式不支持、文件损坏等），输出结构化错误信息。
•	FR-08-04：提供管理员日志查询 API，支持按时间范围、操作类型、任务状态筛选。

5. 非功能性需求
5.1 性能需求
•	NFR-P01：单帧推理延迟（GPU，NVIDIA A100）：AVSegFormer ≤ 50ms/帧，VCT ≤ 80ms/帧，COMBO ≤ 100ms/帧。
•	NFR-P02：系统支持单用户连续处理时长不少于 10 分钟的视频文件（约 15,000 帧 @25fps）。
•	NFR-P03：前端页面首屏加载时间（首次加载）不超过 3 秒（在 100Mbps 内网环境下）。
•	NFR-P04：文件上传接口支持断点续传，大文件上传超时阈值设置为 300 秒。
5.2 可靠性需求
•	NFR-R01：模型推理过程中若发生 OOM（显存溢出），系统应自动捕获异常，向前端返回友好的错误提示，并释放显存资源，不影响系统后续正常使用。
•	NFR-R02：上传的视频文件格式不合规时，系统应在预处理阶段提前检测并拒绝，避免无效推理任务占用 GPU 资源。
•	NFR-R03：任务取消操作应在 5 秒内完成响应，确保后台 GPU 进程与临时文件得到及时清理。
5.3 安全性需求
•	NFR-S01：管理员接口（/api/admin/*）需通过 JWT Token 进行身份验证，Token 有效期为 24 小时。
•	NFR-S02：上传文件需进行格式白名单校验（仅允许 .mp4/.avi/.mov/.mkv），并限制文件大小（≤ 2GB）。
•	NFR-S03：系统日志中不记录用户上传的视频文件原始内容，仅记录文件元信息（大小、格式、时长）。
5.4 可维护性需求
•	NFR-M01：后端算法模块采用策略模式（Strategy Pattern）封装，新增算法只需实现统一接口类，无需修改推理调度核心代码。
•	NFR-M02：前端 Vue 组件按功能划分（upload、player、result、admin），各模块解耦，支持独立测试。
•	NFR-M03：系统配置（模型路径、GPU 设备、端口号等）统一通过 .env 配置文件管理，不硬编码于代码中。
5.5 兼容性需求
•	NFR-C01：前端支持 Chrome 90+、Firefox 88+、Edge 90+ 主流浏览器，分辨率自适应（最低 1280×720）。
•	NFR-C02：后端 Python 版本要求 3.8+，PyTorch 版本要求 2.0+，CUDA 版本要求 11.8+。
•	NFR-C03：系统支持视频格式：MP4（H.264/H.265）、AVI、MOV、MKV，音频格式：AAC、MP3、PCM。

6. 数据需求
6.1 输入数据规范

数据类型	格式要求	处理方式
视频文件	MP4/AVI/MOV/MKV，含音频轨道，帧率 ≥ 15fps	FFmpeg 解封装，帧序列提取
视频帧	RGB，归一化至模型输入尺寸（224×224 或 384×384）	OpenCV resize + 归一化
音频轨道	采样率重采样至 16kHz，单声道/立体声均支持	Librosa 重采样
音频特征	Log-Mel 频谱图，窗口 25ms，步长 10ms，梅尔滤波器 128 个	Librosa feature.melspectrogram

6.2 输出数据规范

数据类型	格式	说明
结果视频	MP4（H.264），含原始音频	分割掩码半透明叠加至原始帧，合成输出
逐帧掩码	PNG，单通道二值图（0/255）	每帧一张，打包为 ZIP 压缩包
彩色掩码视频	MP4（H.264）	掩码区域着色（默认红色 RGBA 叠加）后合成
任务报告	JSON	包含算法名、处理帧数、耗时、平均 J&F 指标

7. 系统架构设计
7.1 前端架构（Vue.js）
前端采用 Vue 3 + Vite 构建，使用 Element Plus 组件库构建 UI，采用 Pinia 管理全局状态，Axios 封装 HTTP 请求，通过原生 WebSocket API 接收后端实时推进度推送。

目录	说明
src/views/	页面级组件：主页（Home）、任务详情（TaskDetail）、管理员后台（Admin）
src/components/	可复用组件：VideoUploader、AlgorithmSelector、ProgressBar、VideoPlayer、ResultViewer
src/store/	Pinia 状态管理：taskStore（任务状态）、algorithmStore（算法列表）
src/api/	Axios 请求封装：taskApi、uploadApi、adminApi
src/utils/	工具函数：格式校验、文件大小格式化、时间格式化
src/assets/	静态资源：样式、图标

7.2 后端架构（Python / FastAPI）
后端采用 FastAPI 框架构建 RESTful API 服务，使用 Uvicorn 作为 ASGI 服务器。模型推理任务通过后台任务（BackgroundTask）或 Celery 异步执行，避免阻塞主进程。

目录/模块	说明
main.py	FastAPI 应用入口，注册路由与中间件
routers/	路由层：upload.py、tasks.py、algorithms.py、admin.py
services/	业务逻辑层：PreprocessService、InferenceService、PostprocessService
models/	算法模型层：base_model.py（接口基类）、avsegformer.py、vct.py、combo.py
schemas/	Pydantic 数据模型：请求/响应 Schema 定义
utils/	工具模块：ffmpeg_utils.py、audio_utils.py、video_utils.py、logger.py
config.py	全局配置：模型路径、GPU 设备、上传目录、日志路径

8. 约束与假设
8.1 技术约束
•	系统部署于配备 NVIDIA GPU（显存 ≥ 16GB，推荐 A100 40GB）的服务器，无 GPU 则无法执行模型推理。
•	前端仅支持 Web 浏览器访问，不提供独立桌面客户端版本。
•	系统当前版本不支持多用户并发推理，同一时刻仅允许一个推理任务占用 GPU 资源。
8.2 业务假设
•	用户上传的视频文件中包含清晰可辨的发声物体，且音视频轨道已同步（无明显 A/V 不同步问题）。
•	系统使用的预训练权重文件已通过 AVSBench 数据集验证，具备基础的泛化能力。
•	管理员用户具备一定的深度学习背景知识，能够理解模型元数据配置的含义。

9. 术语表

术语	全称/解释
AVS	Audio-Visual Segmentation，视频发声物体分割
AVSBench	像素级标注的视听分割基准数据集，包含单声源（S4）、多声源（MS3）及语义子集
AVSegFormer	基于 Transformer 的视听分割算法，采用音频查询机制
VCT	Vision-Centric Transformer，以视觉为中心的视听分割算法
COMBO	基于多阶双边关系的视听分割算法，含双边融合模块（BFM）
J&F	Jaccard 相似系数（J）与 F 度量（F）的均值，AVS 常用评估指标
OOM	Out of Memory，显存溢出错误
Log-Mel	对数梅尔频谱图，音频信号频域特征表示方法
Logits	模型最后一层输出的原始预测分数，经 Sigmoid 后转为概率值
JWT	JSON Web Token，用于用户身份验证的令牌格式
